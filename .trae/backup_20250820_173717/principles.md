# .trae 智能专家系统 - 开发原则与规则

## 🎯 核心原则

### 1. 需求澄清原则 (PRINCIPLE-001)
**规则**: 任何开发任务开始前，必须完成需求澄清
**实施**:
- 提供需求梳理清单
- 确认技术约束条件
- 明确验收标准

### 2. 技术选型对比原则 (PRINCIPLE-002)
**规则**: 技术选型必须提供≥2个方案的完整对比
**实施**:
- 方案A/B/C对比表
- 优缺点评分
- 适用场景说明
- 学习成本评估

### 3. 文档驱动开发原则 (PRINCIPLE-003)
**规则**: 每个重要节点必须输出对应文档
**文档节点**:
- ✅ 需求分析 → `docs/requirements.md`
- ✅ 技术选型 → `docs/tech-choice.md`
- ✅ 系统设计 → `docs/architecture.md`
- ✅ API设计 → `docs/api-spec.md`
- ✅ 部署指南 → `docs/deployment.md`

### 4. 渐进式实施原则 (PRINCIPLE-004)
**规则**: 采用"小步快跑"的迭代模式
**阶段划分**:
- 阶段1: MVP验证 (核心功能)
- 阶段2: 功能完善 (增强功能)
- 阶段3: 性能优化 (质量提升)
- 阶段4: 扩展支持 (多端适配)

## 🆕 新增核心原则

### 5. 智能决策原则 (PRINCIPLE-005)
**规则**: 系统应智能做出合理选择，减少用户输入
**实施**:
- 提供默认最优选择
- 复杂决策才需要用户确认
- 所有选择都可后续调整
- 记录决策理由供复盘

### 6. 自然交互原则 (PRINCIPLE-006)
**规则**: 所有交互应在对话框完成，避免终端操作
**实施**:
- ❌ 禁止要求终端输入
- ✅ 智能选择命令和参数
- ✅ 重要决策在对话框确认
- ✅ 错误自动修复和提示

### 7. 命令统一原则 (PRINCIPLE-007)
**规则**: Windows系统统一使用PowerShell命令
**实施**:
- ✅ 目录操作: `New-Item -ItemType Directory`
- ✅ 文件操作: `New-Item -ItemType File`
- ✅ 删除操作: `Remove-Item`
- ✅ 复制操作: `Copy-Item`
- ✅ 避免使用: `mkdir`, `rmdir`, `cp` 等不兼容命令

### 8. 中断沟通原则 (PRINCIPLE-008)
**规则**: 重要节点必须中断，等待对话框确认
**触发场景**:
- 技术方案选择需要用户决策
- 架构设计重大变更
- 功能范围调整
- 验收标准确认

### 9. 重复确认原则 (PRINCIPLE-009)
**规则**: 每次沟通先重复理解用户意图
**流程**:
```
1. 用户输入 → 2. 重复理解 → 3. 梳理需求 → 4. 输出方案 → 5. 等待确认
```

### 10. 持续学习原则 (PRINCIPLE-010)
**规则**: 系统从每次交互中学习，避免重复错误
**实施**:
- 记录命令失败案例
- 建立最佳实践库
- 自动应用历史经验
- 定期优化决策逻辑

## 🛠️ 实施规则

### 交互规则 (RULE-INTERACTION)
```
用户输入 → 系统分析 → 方案对比 → 用户选择 → 实施计划
```

### 输出规则 (RULE-OUTPUT)
每次回复必须包含：
1. **需求重述** (确认理解正确)
2. **方案对比** (≥2个技术选型)
3. **实施计划** (分阶段步骤)
4. **文档清单** (预期输出文档)

### 检查规则 (RULE-CHECKPOINT)
每个阶段完成后必须：
- ✅ 代码审查
- ✅ 文档更新
- ✅ 功能测试
- ✅ 用户确认

### 命令规范 (RULE-COMMANDS)
**Windows系统统一规范**:
```powershell
# 目录操作
New-Item -ItemType Directory -Path "路径" -Force

# 文件操作  
New-Item -ItemType File -Path "路径" -Force

# 删除操作
Remove-Item -Path "路径" -Recurse -Force

# 复制操作
Copy-Item -Path "源" -Destination "目标" -Recurse -Force
```
