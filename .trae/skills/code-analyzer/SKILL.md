# 代码分析器

## 描述
这个技能用于分析项目的代码结构，包括项目类型检测、技术栈识别、代码质量评估等。

## 使用场景
- 新项目启动时，需要了解当前项目的技术栈
- 接手别人的项目时，快速了解项目结构
- 技术选型前，需要了解现有项目的架构
- 定期进行项目健康度检查和代码质量评估

## 指令

当用户使用这个技能时，你应该：

1. **检测项目类型**
   - 检查 `package.json`（Node.js）、`requirements.txt`（Python）、`go.mod`（Go）等文件
   - 分析技术栈（Vue、React、FastAPI、Django 等）

2. **分析项目结构**
   - 列出主要目录和关键文件
   - 检测依赖关系
   - 识别核心业务模块

3. **提供改进建议**
   - 代码结构优化建议
   - 潜在问题提示
   - 最佳实践推荐

4. **输出格式**
   - 结构化的分析报告
   - 清晰的分类和重点高亮
   - 可操作的改进建议

## 示例

### 输入
"分析一下这个项目的代码结构"

### 输出
```
# 项目代码分析报告

## 项目类型
- Vue 3 + Vite + TypeScript

## 技术栈
- 前端：Vue 3, TypeScript, Pinia
- 构建工具：Vite, ESLint, Prettier
- 测试：Vitest, Testing Library

## 建议
1. 添加单元测试覆盖核心业务逻辑
2. 补充 API 文档
3. 优化项目结构，分离组件和工具
```
