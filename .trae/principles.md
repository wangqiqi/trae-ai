## 🎯 原则详解（每个30秒掌握）

### P-001 需求澄清原则 📋
**核心公式**：需求 = 功能清单 + 技术约束 + 验收标准

**🎯 一句话模板**:
> "我要做一个[项目类型]，需要[核心功能]，技术用[技术栈]，预期[完成时间]"

**📊 需求检查表**:
- ✅ 功能边界明确
- ✅ 技术约束清晰  
- ✅ 验收标准量化
- ✅ 时间节点确定

**💡 智能体调用**:
```
@产品经理 我要做一个Vue3任务管理系统，需要用户登录和任务CRUD，技术用Vue3+FastAPI，预期2周完成
```

### P-002 技术选型原则 ⚖️
**决策矩阵**（复制即用）：
| 维度 | Vue3 | React | Angular |
|------|------|-------|---------|
| 学习成本 | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| 开发效率 | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ |
| 社区生态 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| **推荐指数** | **95%** | **90%** | **70%** |

**🤖 智能决策**：复杂场景才询问，简单场景直接推荐

### P-006 高效沟通原则 💬
**沟通公式**：结构化提问 = 背景 + 目标 + 约束

**🎯 标准格式**:
```
【背景】当前在做Vue3项目
【目标】需要实现用户认证功能
【约束】使用JWT，要求2小时内完成
```

### P-007 时间信息原则 ⏰
**每次对话必备**:
- 📅 当前日期：`2025年8月更新`
- 👤 用户信息：`git config user.name`
- 📊 项目时间：`项目创建时间 + 预计周期`

**💡 应用场景**：
- 文档版权自动更新
- 项目里程碑提醒
- 代码提交时间记录

### P-008 结构化沟通原则 🎯
**核心流程**：理解 → 思考 → 方案 → 执行

**🎯 沟通公式**：
1. **理解复述**："我理解你的需求是..."
2. **思考展示**："我的思考过程是..."
3. **方案提供**："解决方案有A/B两种..."
4. **执行决策**："无需决策直接执行"或"需要你选择..."

**📋 标准模板**:
```
🎯 **理解你的需求**：你想要[重新组织的需求描述]

💭 **我的思考**：基于[分析依据]，我认为[核心思路]

🔧 **解决方案**：
- 方案A：[描述] - 优点：[...] - 缺点：[...]
- 方案B：[描述] - 优点：[...] - 缺点：[...]

⚡ **执行建议**：[直接执行/需要你决策]
```

### P-009 测试优先原则 🧪
**核心思想**：测试先行，质量保证

**🎯 实施步骤**：
1. **测试设计**：@测试工程师 设计测试用例
2. **代码开发**：基于测试用例编码
3. **验证反馈**：测试报告驱动优化

**📊 测试金字塔**：
- 单元测试：80%覆盖率
- 集成测试：关键路径覆盖
- E2E测试：用户场景验证

**💡 一键测试**：
```bash
# 生成测试用例
@测试工程师 为Vue3登录功能编写测试用例

# 执行测试
@测试工程师 运行项目单元测试

# 测试报告
@测试工程师 生成测试覆盖率报告
```

### P-010 安全优先原则 🔒
**核心理念**：安全不是可选项，是必选项

**🛡️ 安全检查表**：
- ✅ 输入验证和消毒
- ✅ 身份认证和授权
- ✅ 数据加密传输
- ✅ 敏感信息脱敏
- ✅ 安全头配置

**🔧 智能体调用**：
```bash
# 安全审查
@DevOps工程师 检查项目安全配置

# 认证实现
@Python工程师 实现JWT用户认证

# 数据保护
@系统架构师 设计数据加密方案
```

### P-011 性能优化原则 ⚡
**优化策略**：先测量，再优化

**📊 性能指标**：
- **前端**：首屏加载 < 2秒
- **后端**：API响应 < 200ms
- **数据库**：查询 < 100ms
- **移动端**：包大小 < 5MB

**🔧 优化工具**：
```bash
# 性能分析
@系统架构师 分析Vue3应用性能瓶颈

# 代码优化
@前端工程师 优化React组件渲染性能

# 部署优化
@DevOps工程师 配置CDN和缓存策略
```

### P-012 可持续维护原则 🔄
**长期视角**：今天写的代码，6个月后还能维护

**🎯 维护标准**：
- 📋 **代码规范**：统一编码风格
- 🧪 **测试覆盖**：关键功能100%测试
- 📚 **文档完整**：README + API文档
- 🔄 **依赖更新**：定期安全检查
- 🗂️ **版本管理**：语义化版本控制

**💡 维护检查**：
```bash
# 代码质量检查
@技术文档工程师 生成项目维护指南

# 依赖更新
@系统架构师 检查项目依赖安全更新

# 文档同步
@技术文档工程师 更新API文档
```

**💡 实际应用**：
- 复杂需求：先复述确认，再提供选项
- 简单需求：直接理解执行
- 模糊需求：主动提问澄清

## ⚠️ 常见误区与纠正

### 🚫 原则应用误区

| 误区 | 纠正方法 | 应用原则 |
|------|----------|----------|
| 只关注功能，忽略安全 | 每个功能都问："这安全吗？" | P-010 |
| 开发完才写测试 | 测试先行，功能后补 | P-009 |
| 文档最后才写 | 开发过程中实时生成 | P-005 |
| 技术选型只看流行度 | 全面评估6个维度 | P-002 |
| 需求理解有偏差 | 使用结构化沟通模板 | P-008 |

### 🎯 快速决策工具

**技术选型决策树**（30秒内）：
```
问题：React vs Vue3？
→ 团队熟悉度？React高→选React，Vue3高→选Vue3
→ 项目复杂度？高→React，中低→Vue3
→ 性能要求？极高→Vue3，一般→两者皆可
→ 学习成本？低→Vue3，可接受→React
```

**功能优先级决策**（2分钟内）：
```
标准：MVP → 增强 → 优化 → 扩展
问题：用户注册功能优先级？
→ MVP：基本注册登录 ✓
→ 增强：邮箱验证 ✓
→ 优化：社交登录 ✓
→ 扩展：多因子认证 ✓
```
