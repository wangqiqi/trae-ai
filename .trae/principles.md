## 🎯 原则详解（每个30秒掌握）

### P-001 需求澄清原则 📋
**核心公式**：需求 = 功能清单 + 技术约束 + 验收标准

**🎯 一句话模板**:
> "我要做一个[项目类型]，需要[核心功能]，技术用[技术栈]，预期[完成时间]"

**📊 需求检查表**:
- ✅ 功能边界明确
- ✅ 技术约束清晰  
- ✅ 验收标准量化
- ✅ 时间节点确定

**💡 智能体调用**:
```
@产品经理 我要做一个Vue3任务管理系统，需要用户登录和任务CRUD，技术用Vue3+FastAPI，预期2周完成
```

### P-002 技术选型原则 ⚖️
**决策矩阵**（复制即用）：
| 维度 | Vue3 | React | Angular |
|------|------|-------|---------|
| 学习成本 | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐ |
| 开发效率 | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ |
| 社区生态 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| **推荐指数** | **95%** | **90%** | **70%** |

**🤖 智能决策**：复杂场景才询问，简单场景直接推荐

### P-003 文档驱动原则 📝
**黄金法则**：代码未动，文档先行

**📁 自动生成文档结构**:
```
docs/
├── requirements.md   # @产品经理生成
├── architecture.md # @系统架构师生成  
├── api-spec.md     # @后端工程师生成
├── deployment.md   # @DevOps工程师生成
└── checklist.md    # @测试工程师生成
```

### P-004 渐进实施原则 📈
**4D开发模型**：
| 阶段 | 用时 | 交付物 | 智能体分工 |
|------|------|--------|------------|
| **Day 1-2** | 2天 | MVP核心 | @产品经理 + @前端工程师 |
| **Day 3-5** | 3天 | 功能完善 | @前端 + @后端工程师 |
| **Day 6** | 1天 | 性能优化 | @系统架构师 + @测试工程师 |
| **Day 7-8** | 2天 | 多端适配 | @DevOps + @部署工程师 |

### P-005 自动文档原则 🤖
**一键生成**：
```bash
# 开发过程中自动更新文档
python .trae/workflows/trae-console.py "生成API文档"

# 代码变更后自动同步
@技术文档工程师 更新API文档
```

### P-006 高效沟通原则 💬
**沟通公式**：结构化提问 = 背景 + 目标 + 约束

**🎯 标准格式**:
```
【背景】当前在做Vue3项目
【目标】需要实现用户认证功能
【约束】使用JWT，要求2小时内完成
```

### P-007 时间信息原则 ⏰
**每次对话必备**:
- 📅 当前日期：`2025年8月更新`
- 👤 用户信息：`git config user.name`
- 📊 项目时间：`项目创建时间 + 预计周期`

**💡 应用场景**：
- 文档版权自动更新
- 项目里程碑提醒
- 代码提交时间记录
