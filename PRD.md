# EasyMan 产品需求文档 (PRD)

## 📋 产品概述

**产品名称**: EasyMan (Easy Manager)
**产品定位**: 纯P2P去中心化敏捷看板系统
**核心价值**: 无需服务器，支持离线协作的轻量级敏捷开发工具

## 🎯 核心功能需求

### MVP版本功能清单

#### 1. 基础看板功能
- **三列布局**: TODO → DOING → DONE
- **任务卡片**: 包含标题、描述、优先级、创建时间
- **拖拽排序**: 卡片可在列内和列间自由拖拽
- **添加任务**: 快速创建新任务
- **删除任务**: 支持删除任务
- **编辑任务**: 支持修改任务内容

#### 2. 数据存储
- **本地存储**: 使用浏览器IndexedDB存储数据
- **P2P同步**: 通过WebRTC实现多设备实时同步
- **离线支持**: 断网后可正常使用，联网后自动同步

#### 3. 用户界面
- **响应式设计**: 支持桌面和移动端
- **简洁美观**: 现代化UI设计
- **深色模式**: 支持明暗主题切换

## 🎨 用户故事

### 用户故事1：创建任务
```
作为：敏捷开发团队成员
我想：快速创建一个新的任务卡片
所以：我可以将新需求添加到TODO列表
```

所以：我可以直观地管理任务进度

```

```

## 📐 技术规格

### 前端技术栈
- **框架**: React 18 + TypeScript
- **状态管理**: Zustand
- **拖拽**: React-DnD
- **样式**: CSS Modules
- **构建**: Vite

### P2P技术
- **通信**: WebRTC DataChannel
- **信令**: Gun.js
- **存储**: IndexedDB + localStorage

### 数据模型

```typescript
interface Task {
  id: string
  title: string
  description: string
  priority: 'low' | 'medium' | 'high'
  status: 'todo' | 'doing' | 'done'
  createdAt: Date
  updatedAt: Date
  assignee?: string
}

interface BoardState {
  tasks: Task[]
  columns: {
    todo: Task[]
    doing: Task[]
    done: Task[]
  }
}
```

## 🎨 界面设计

### 布局结构
```
┌─────────────────────────────────────────────┐
│  EasyMan - 纯P2P敏捷看板                    │
├─────────────────────────────────────────────┤
│  [+ 添加任务]  [主题切换]  [P2P状态]         │
├─────────────────────────────────────────────┤
│  ┌───────────┐  ┌───────────┐  ┌───────────┐ │
│  │   TODO    │  │  DOING    │  │   DONE    │ │
│  │           │  │           │  │           │ │
│  │ 任务卡片1  │  │ 任务卡片2 │  │ 任务卡片3 │ │
│  │ 任务卡片4  │  │           │  │           │ │
│  └───────────┘  └───────────┘  └───────────┘ │
└─────────────────────────────────────────────┘
```

### 任务卡片设计
- **标题**: 粗体显示，最多50字符
- **描述**: 灰色小字，最多200字符
- **优先级**: 颜色标识（红=高，黄=中，绿=低）
- **时间**: 显示创建时间
